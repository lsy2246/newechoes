# styles

## 职责

站点样式与主题配置（src/styles，Tailwind）。

## 接口定义（可选）

无对外公共 API。

## 行为规范

### 样式调整
**条件**: 需要调整主题或排版。
**行为**: 修改样式文件并确保与组件/页面一致。
**结果**: 页面样式符合预期。

### 图谱入口与侧栏卡片样式
**条件**: 站点增加全局图谱弹窗入口或文章侧栏增加关系信息卡片。
**行为**: 在 Header 和文章样式文件中补充图标按钮、双栏弹窗、目录树、可拖拽节点图、反向链接列表等配套样式，并同步深浅色主题表现。
**结果**: 新增的知识站入口与关系区块与现有站点视觉语言保持一致。

### 首页编辑器画布与屏幕投影过渡
**条件**: 首页 `HomeDiorama` 需要同时承担首屏编辑器展示和 3D 房间揭示。
**行为**: `diorama.css` 使用 CSS 变量驱动首屏画布的位置、尺寸、圆角和舞台缩放，让无限笔记编辑器从全屏单画面逐步收束到笔记本屏幕投影；同时用独立的场景透明度变量控制 3D 房间露出时机。
**结果**: 首页过渡呈现“嵌套观看 / 二层叙事”效果，而不是双层画布叠加。

### 首页竖屏命中层与手机载体协同
**条件**: 首页在纵向视口下进入 3D 揭示阶段。
**行为**: `diorama.css` 继续只为当前投影出的屏幕区域提供 invisible hitarea，不承担第二层可见编辑器；视觉上的手机载体完全由 three 场景绘制。
**结果**: 竖屏首页仍保持单一内容源，过渡到 3D 后不会出现额外的桌面电脑外壳、键盘层或中间态设备语义。
