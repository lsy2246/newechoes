# components

## 职责

可复用 UI 组件（src/components）。

## 接口定义（可选）

无对外公共 API（内部组件为主）。

## 行为规范

### 组件复用
**条件**: 页面需要复用 UI 片段。
**行为**: 通过 props/slots 组合组件。
**结果**: 统一外观与交互体验。

### Countdown 字号语义化
**条件**: 页面渲染倒计时或展示型数字。
**行为**: `Countdown` 组件通过 `countdown-row`、`countdown-unit`、`countdown-value`、`countdown-label`、`countdown-expired` 等语义类接入全站 typography token，不在组件内使用 Tailwind `text-*` 直接定义孤立字号。
**结果**: about 页倒计时数字、标签与全站标题/辅助文字阶梯一致。

### WASM 计算下沉
**条件**: Search / ArticleFilter 需要执行检索或筛选。
**行为**: 通过 lib/wasmWorkerClient 与 Worker 通信，异步获取结果。
**结果**: 主线程渲染与交互更顺畅。

### 搜索实例隔离
**条件**: 同时渲染多个 Search 实例（桌面/移动）。
**行为**: 组件内部使用实例容器判断 click-outside，避免跨实例干扰；索引初始化去重。
**结果**: 交互互不影响，避免重复初始化开销。

### 主题切换轻量化
**条件**: 页面存在多个 ThemeToggle 实例或频繁切换主题。
**行为**: 复用过渡资源与波纹节点，初始化去重，减少样式重算与 DOM churn。
**结果**: 动画效果一致的前提下切换更顺滑。

### 三维组件按需加载
**条件**: 首次进入 WorldHeatmap。
**行为**: three 运行时动态加载后再初始化场景。
**结果**: 首屏包体更小，加载更平滑。

### ArticleFilter 筛选去闪烁
**条件**: 快速切换标签或无标签清除筛选。
**行为**: 标签加载与筛选加载状态分离，筛选加载不影响标签占位与加载指示，无标签清除直接跳过筛选。
**结果**: 标签占位提示稳定，加载指示不闪烁。

### 全局图谱入口弹窗
**条件**: 用户在桌面端或移动端头部点击图谱入口按钮。
**行为**: 通过独立的 GlobalGraphModal 组件打开双栏弹窗，左侧展示全站目录树并高亮当前位置，右侧展示可拖拽节点图谱。
**结果**: 站点具备统一的全局图谱入口，同时兼顾目录式浏览和节点式探索。

### 相册预览滚动恢复
**条件**: 用户在 `PhotoAlbumMasonry` 中滚动浏览后打开大图预览。
**行为**: 组件记录打开预览时的页面滚动位置，预览期间锁定背景滚动，关闭后恢复到原位置。
**结果**: 用户返回原浏览位置，不会被带回页面顶部。

### 相册时间顺序稳定化
**条件**: `PhotoAlbumMasonry` 合并 Google Photos 分页结果后渲染瀑布流。
**行为**: 组件按 `takenAt` 对已加载照片做统一时间倒序排序，并让瀑布流和预览共用排序后的数组。
**结果**: 相册展示顺序稳定贴近 Google Photos 官方分享页时间顺序。

### Git 项目按热度排序
**条件**: `GitProjectCollection` 获取到项目列表并准备渲染卡片。
**行为**: 组件优先按 `stars` 倒序排序，星标相同时再按 `forks` 和最近更新时间稳定排序。
**结果**: 项目列表优先展示更受欢迎的仓库。

### 布局按平台启用专属能力
**条件**: 共享布局在不同部署平台渲染。
**行为**: `Layout.astro` 通过平台能力层提供的统一 observability 配置决定是否启用 `Vercel Speed Insights`，并消费 `Cloudflare / EdgeOne` 的监控脚本插槽，而不是在组件内散写平台判断。
**结果**: 平台专属能力不会污染其他运行环境，后续继续补平台埋点或监控时只需要扩展平台层出口。

### 首页单一可见源与分端载体
**条件**: 用户访问首页并滚动浏览 `HomeDiorama`，或浏览器在横向/纵向视口之间切换。
**行为**: 组件仅保留一层可见的 screen texture 作为首页内容源；交互拖拽通过投影到当前屏幕区域的 invisible hitarea 完成。场景与 board 版式不再按桌面/平板/手机宽度断点划分，而是按画幅比例二分：横屏使用电脑载体与横向 board，竖屏使用手机载体与纵向 board，并在横竖切换时重建场景而不切换内容数据源。
**结果**: 首页首屏与 3D 场景内屏幕始终一致，同时避免出现“电脑里显示手机页面”、双画布错位，或平板这类中间态带来的叙事比例不统一问题。
