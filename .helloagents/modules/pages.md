# pages

## 职责

定义站点路由与页面结构（src/pages，Astro 路由）。

## 接口定义（可选）

无对外公共 API。

## 行为规范

### 页面渲染
**条件**: 访问路由或进行静态构建。
**行为**: 组合 components、styles 与内容数据。
**结果**: 生成可访问的静态页面。

### Git 项目 API 分页
**条件**: 访问 `/api/git-projects` 获取 GitHub 项目。
**行为**: 基于 GitHub Link header 解析是否有下一页与总页数。
**结果**: 分页准确，避免出现空的第二页。

### 文章页关系侧栏
**条件**: 访问文章详情页且右侧目录面板可见。
**行为**: 页面基于现有文章正文中的站内链接路径，识别哪些文章引用了当前文章，并将结果渲染到目录面板下方。
**结果**: 文章关系信息从正文区域移出，集中到侧栏辅助区展示。

### 文章目录桌面断点
**条件**: 用户在常用桌面宽度访问文章详情页。
**行为**: 目录侧栏从 `xl` 断点开始展示，而不是仅在超宽屏可见。
**结果**: 文章目录和反向链接在更常见的桌面视口下可见。

### 首页二层叙事入口
**条件**: 用户访问 `/` 首页，或在浏览过程中切换横向/纵向视口。
**行为**: `index.astro` 继续通过 `HomeDiorama` 承载首页；首屏先显示完整的无限笔记编辑器工作台，滚动后再揭示其属于 3D 场景中的屏幕。场景与内容布局统一按比例二分：横屏揭示为电脑，竖屏揭示为手机，首页内容仍来自同一份 board 数据。
**结果**: 首页在不同视口比例下都保留 3D 叙事，同时确保首屏内容比例与外层载体逻辑天然一致。
