# 404 Page Design

## Goal

Replace the current 404 page with a quiet black-and-white illustration layout close to the provided reference image: a centered hand-drawn character illustration, simple "404 Not Found" messaging, and a clear return-home action.

## Scope

- Use `D:\Downloads\c.svg` as the primary illustration asset.
- Convert the SVG into a theme-aware inline Astro component instead of using it as an image file.
- Redesign `src/pages/404.astro` around a minimal centered layout.
- Keep the site header for navigation continuity.
- Hide the footer on the 404 page so the composition stays close to the reference.

## Visual Design

The page should feel like a personal blog error page rather than a product landing page. The composition is vertically centered with generous negative space. The illustration sits above the text, followed by:

- Primary title: `404 Not Found`
- Supporting copy in Chinese explaining that the page may not exist or may have moved.
- One primary button: `返回首页`

The layout should avoid decorative cards, gradients, complex backgrounds, and colorful accents. The page should remain monochrome in both themes.

## Illustration Theming

The source SVG currently contains two hard-coded fill groups:

- `#000000` for the character and line work.
- `#ffffff` for the background and white details.

The implementation should convert these fills to CSS variables:

- `--not-found-ink`
- `--not-found-paper`

In light mode:

- `--not-found-ink: #050505`
- `--not-found-paper: #ffffff`

In dark mode:

- `--not-found-ink: #f5f5f5`
- `--not-found-paper: #000000`

This keeps the same drawing while avoiding a bright white SVG rectangle in dark mode.

## Component Plan

- Add a focused illustration component, such as `src/components/NotFoundIllustration.astro`.
- Keep the large SVG markup out of `404.astro`.
- Use CSS classes or SVG group attributes so theme colors are controlled by the page.
- Preserve the original SVG viewBox and aspect ratio.

## Responsive Behavior

- Desktop: illustration centered above the text, with enough vertical air to resemble the reference.
- Tablet/mobile: illustration scales down without cropping; text and button remain centered.
- The page must not create horizontal scrolling at 375px width.
- Text must not overlap the illustration or button.

## Accessibility

- The visible heading should identify the page state.
- The illustration is decorative and should be hidden from assistive technology.
- The return-home link should be keyboard focusable with a visible focus state.
- Color contrast must remain readable in both light and dark themes.

## Out Of Scope

- Adding search suggestions or article recommendations.
- Creating multiple illustration assets.
